# SPEC-grag Project Files

This directory stores project-local SPEC-grag configuration and runtime state.

- `config.toml` is the committed project configuration.
- `graph/` is generated by `/spec-core` and should stay out of git.
- `runs/` contains optional request/result artifacts when `run.save_artifacts`
  is enabled.
- `pending/` contains guarded Concept diff proposals that should be resolved
  with `/spec-core --accept`, `--reject`, `--revise`, and `--apply`.

The normal template is production-oriented: it expects schema LLM extraction,
LLM classification, LLM answer generation, LLM Concept diff proposals, and real
embeddings. Configure `[llm].provider`, Codex / Claude command, model, effort,
and Ollama or equivalent embedding settings before running a production config.
The stage tables inherit the selected `[llm]` provider so switching between
Codex and Claude does not require editing every LLM-backed stage. Claude's
default is the full Claude Code model name `claude-sonnet-4-6`; short aliases
such as `sonnet` are accepted by Claude Code but are not pinned.

For CI or fresh-install command wiring checks, generate a separate smoke config
with `scripts/setup_project.py --smoke`. If this file was installed with a
smoke config, treat that project as a wiring check only. Smoke mode is not a
quality profile and should not be used to evaluate GraphRAG behavior.
