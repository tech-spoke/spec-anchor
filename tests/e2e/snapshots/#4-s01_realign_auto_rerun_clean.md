# #4-s01 答案なし呼び出し → 自動再実行 → 整形済み RealignResult

利用者が `/spec-realign "ログイン失敗時のロック仕様を決めたい"` を **答案なしで** 呼んだ場合。Agent は内部で Agentic Search → 答案構成 → 再実行を行い、利用者には整形済みの 4 区分だけを見せる。内部信号 (答案待ちの停止) は一切表に出さない。

---

今回守る制約
  - ログイン失敗回数のしきい値超過でアカウントを一時ロックする。
    根拠の種類: Source Specs
    参照: docs/spec/auth.md#0006-account-lockout
    適用範囲: ログイン失敗時のロック処理を実装・変更する作業

今回扱う修正候補または検討対象
  - 失敗回数のカウント単位 (IP 単位かアカウント単位か) を確定する
  - ロック解除の経路 (時間経過 / 管理者解除) を決める

競合 / 不確実性 / 人間レビューが必要な点
  - ロック時間の具体値は仕様資料に明記がなく、人間確認が必要

課題プロンプトへの回答または修正案
  - 連続ログイン失敗が規定回数を超えたらアカウントを一時ロックする方針とする。カウント単位とロック時間は上記の人間確認を経て確定する。
