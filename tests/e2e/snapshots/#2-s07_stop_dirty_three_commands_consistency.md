# #2-s07 3 コマンド一貫性: ③ 保持物の更新が必要

同じ「Source Specs が変更され保持物が古い」状態に対し、3 コマンドが同一の ③ フォーマットで停止を表示することを確認するシナリオ。

---

【/spec-inject "認証フローの制約" を呼んだ場合】

■ 保持物の更新が必要です

  Source Specs が変更され、保持している情報が古くなっています。この状態では今回の制約生成へ進みません。

  次の操作:
    /spec-core を実行して保持物を更新してください。

【/spec-realign "認証フローを直したい" を呼んだ場合】

■ 保持物の更新が必要です

  Source Specs が変更され、保持している情報が古くなっています。この状態では回答整形へ進みません。

  次の操作:
    /spec-core を実行して保持物を更新してください。

【/spec-core を呼んだ場合】

`/spec-core` は保持物を更新するコマンドなので、この「保持物の更新が必要」状態は `/spec-core` 自身の実行で解消される (停止ではなく更新が走る)。更新後に正常完了フォーマット (「保持物の更新が完了しました」) を表示する。

一貫性のポイント: ③ の見出し「■ 保持物の更新が必要です」と次の操作「/spec-core を実行してください」は `/spec-inject` と `/spec-realign` で完全に同一。`/spec-core` はこの状態の解決側になる。
