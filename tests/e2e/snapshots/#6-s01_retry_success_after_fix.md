# #6-s01 構造化失敗 → CLI error 詳細で修正 → 2 回目で成功

利用者が `/spec-realign "失敗ログインのロック仕様"` を呼び、Agent が組み立てた最初の答案が形式不備で停止した。Agent は CLI が示した不備箇所 (回答/修正案セクションが空) だけを直して 1 回再実行し、成功した。利用者には最終の整形済み RealignResult だけが見える (1 回目の失敗とリトライ過程は表に出さない)。

---

今回守る制約
  - 連続ログイン失敗がしきい値を超えたらアカウントを一時ロックする。
    根拠の種類: Source Specs
    参照: docs/spec/auth.md#0006-account-lockout
    適用範囲: ログイン失敗時のロック処理を実装・変更する作業

今回扱う修正候補または検討対象
  - 失敗回数のカウント単位 (IP / アカウント) を確定する

競合 / 不確実性 / 人間レビューが必要な点
  - ロック時間の具体値は資料に明記がなく、人間確認が必要

課題プロンプトへの回答または修正案
  - 連続ログイン失敗が規定回数を超えたらアカウントを一時ロックする。カウント単位とロック時間は人間確認を経て確定する。
