# SPEC-grag Project Files

This directory stores project-local SPEC-grag configuration and runtime state.

- `config.toml` is the committed project configuration. In this repository it is
  intentionally production-oriented.
- `config.smoke.toml` preserves the old no-dependency smoke configuration for
  reference and test fixture updates. Normal commands do not load it
  automatically.
- `graph/` is generated by `/spec-core` and should stay out of git.
- `runs/` contains optional request/result artifacts when `run.save_artifacts`
  is enabled.
- `pending/` contains guarded Concept diff proposals that should be resolved
  with `/spec-core --accept`, `--reject`, `--revise`, and `--apply`.

Production-quality runs use schema LLM extraction, LLM classification, LLM
answer generation, LLM Concept diff proposals, and real embeddings. This local
checkout uses the project-level `[llm] provider = "codex_cli"` switch for all
LLM-backed stages, inherits provider `effort = "low"`, and points embedding at
Ollama `bge-m3` (`dimension = 1024`), the recommended default for Japanese /
multilingual specification documents.
Switching to Claude should be done by changing `[llm].provider` to
`claude_cli`, not by editing every stage table. The Claude default uses the
full Claude Code model name `claude-sonnet-4-6`; aliases such as `sonnet` are
valid for Claude Code but are not pinned.

There is no separate `local` profile; local execution is production execution
with local or CLI-backed providers. Run a small temporary-project probe before a
full self-project production run, because schema extraction over every file in
`テスト用ドキュメント/` can consume a non-trivial amount of LLM tokens.
If existing generated files under `.spec-grag/graph/` were produced by smoke
runs, rebuild them with `spec-core --all` after the token probe so their
embedding metadata matches `bge-m3`.

Smoke settings use deterministic extraction, template answers, rule-based
classification, source-derived Concept diffs, or `stable_hash` embeddings only
to verify command wiring. They are allowed only through explicit smoke paths
such as `SPEC_GRAG_SMOKE=1`, `scripts/setup_project.py --smoke`, or test
fixtures.
