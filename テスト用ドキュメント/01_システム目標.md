# システム目標

## 1. Core の保全とカスタマイズ性の確保

Core は upstream / git フォーク管理対象であり、Customize は利用者カスタマイズ領域である。自由度を確保しつつ Core を保全する。

- この構造のなかで、完全なカスタマイズ性を確保する
- カスタマイズしたい粒度に沿ったカスタマイズ方法を提供する

## 2. 実装者によらず書き方が統一されること

LLM や人が実装しても書き方が変わらないように制限する。

- 「これを触るときはここに書く」という MAP が明確であること
- 書き方のバリエーションが少なく、出力が収束すること

## 3. Core はシンプルに、学習は少なく

パッケージとして配付するため、Core に触れる開発者の参入障壁を下げる。

- Core のソースはできるだけシンプルにする
- Core がシンプルであれば Customize もしやすい
- フレームワーク内部がシンプルである必要はない

## 4. TypeScript 統一の活用

サーバーとフロントが同じ TypeScript であることを活かす。

- Zod schema / 型定義をサーバーとクライアントで共有する
- バリデーション・契約・DTO を一箇所で定義し、両側から参照する
