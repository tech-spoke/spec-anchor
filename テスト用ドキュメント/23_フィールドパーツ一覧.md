# フィールドパーツ一覧

## 目的

管理画面 UI で Core が提供するフィールドパーツ群を一覧化し、各パーツの実装に使う推奨ライブラリを示す。

カスタマイズ粒度に沿った手段を提供する支配原則（doc20）に従い、customize 契約が成立する実質的な単位として **振る舞い分類 16** を定める。16 分類は doc26 の `CommonFieldProps` / パーツ固有 props / customize 機構（patchComponent / replaceComponent）が扱う単位と一致する。

### スコープ

この文書で扱うもの:

- 管理画面 UI に並ぶフィールドパーツの列挙（用途例）
- パーツの振る舞い分類（16 分類）
- ベースレイヤー（UI プリミティブ / フォーム管理）の採用ライブラリ
- 16 分類ごとの推奨ライブラリとカスタマイズ契約の接続点

この文書で扱わないもの:

- 各パーツの props 定義（doc26 「パーツ固有の props」）
- `CommonFieldProps` / 3 層構造 / RHF 接続（doc26）
- 基本構造（doc24）・配置操作（doc25）
- customize 手段の全体像（doc20）
- customize の進行と分類ごとの契約相性（doc26.5）

## 振る舞い分類（16 分類）

UI 振る舞いが異なる単位で分類したもの。customize 契約が成立する実質的な単位。

| # | 分類 | 含まれるパーツ | 用途例 |
|---|---|---|---|
| 1 | 単一値入力 | text / number / email / password / url / phone / textarea / hidden | 商品名、SKU、価格、在庫数、メタタイトル、メタディスクリプション、内部 ID |
| 2 | 選択 | select / tree-select / radio / checkbox（単体/複数）/ toggle / toggle group / tags | カテゴリ、ステータス、通貨、国、性別、規約同意、権限、商品タグ |
| 3 | 日付/時刻 | date picker / datetime picker / date range | 生年月日、発売日、公開開始日時、セール期間、クーポン有効期間 |
| 4 | リッチテキスト | WYSIWYG / markdown / code editor | 商品説明、ブランド説明、開発者向け説明、カスタム CSS/JS、メールテンプレート |
| 5 | メディアアップロード | 画像（単体/複数並び替え）/ ファイル | ブランドロゴ、OGP 画像、商品画像ギャラリー、PDF 仕様書、CSV 一括取込 |
| 6 | カラーピッカー | color / icon picker | タグ色、テーマカラー、カテゴリアイコン、メニューアイコン |
| 7 | リピーター（動的行）| 行の追加・削除・並び替え | 注文明細、バリエーション、複数配送先 |
| 8 | キーバリュー | 任意キーと値のペア | 商品メタ情報、カスタム属性 |
| 9 | テーブル（インライン編集）| カラム定義 + 行内編集 | 在庫数一括編集、価格マトリクス |
| 10 | 表示専用 | placeholder / view / 読み取り専用 / 住所フォーム（郵便番号→住所自動入力）| 注文番号、作成日時、在庫ステータス、配送先住所の参照表示 |
| 11 | チャート | 折れ線 / 棒 / 円 / 面 / 複合 / スパークライン / ヒートマップ / ファネル / ランキング | 売上推移、月別売上比較、売上構成比、在庫推移、売上×注文件数、カート転換率 |
| 12 | 統計カード（KPI）| 数値カード / ゲージ / プログレス | 本日売上、注文数、客単価、在庫総数、目標達成率、在庫消化率 |
| 13 | 通知・アラート | トースト / バナー / バッジ / カウンター | 保存成功、エラー通知、在庫アラート、メンテナンス告知、未処理注文数 |
| 14 | オーバーレイ | モーダル / ドロワー / ポップオーバー / ツールチップ / コンテキストメニュー | 削除確認、カテゴリ編集、注文詳細サイド表示、ヘルプ表示、行の右クリック |
| 15 | コマンド系 | コマンドパレット | グローバル検索、クイックアクション |
| 16 | ツリー操作 | 階層構造の並び替え・階層変更・親子一括移動 | カテゴリツリー、メニューツリー、コンテンツ階層、権限ツリー |

### form value の有無

1〜9 は form value を持つフィールド（RHF `FormField` を介して form と接続）。10〜16 は form value を持たず、Store 読み取り / Action dispatch / UI 制御を担う（詳細な接続方式は doc26 参照）。

EC 特有の派生（金額・割合・重量・寸法）は 1 単一値入力の派生、バリエーションマトリクスは 9 インラインテーブルの派生として扱う。住所フォーム（郵便番号→住所自動入力）は内部に複数入力要素を持つが form 上は単一 field として扱うため、表示・整形の観点から 10 表示専用の近縁として位置づけ、詳細な props 定義は doc26 の `AddressFieldProps` を参照する。

## ベースレイヤー

| レイヤー | 採用 | 理由 |
|---|---|---|
| UI プリミティブ | shadcn/ui（Radix / Base UI + Tailwind）| コピペ方式で所有権が core 側にくる。ロックインなし |
| フォーム管理 | React Hook Form + Zod | 事実上の標準。uncontrolled で高パフォーマンス。resolver で Zod と接続 |

詳細な接続規約（`FormField` / `useController` / `useFieldArray` / `zodResolver` 等）は doc26 で定める。

## 16 分類ごとの推奨ライブラリ

| # | 分類 | 推奨 | 備考 |
|---|---|---|---|
| 1 | 単一値入力 | shadcn/ui Input | text / number / email / password / url は Input 派生で済む |
| 2 | 選択 | shadcn/ui Select / Combobox | ツリーセレクトは Command (cmdk) の再帰拡張で対応 |
| 3 | 日付/時刻 | react-day-picker v9 | shadcn/ui の Calendar が内部で採用 |
| 4 | リッチテキスト | Tiptap | ProseMirror ベース。プラグイン拡張性が最も高い |
| 5 | メディアアップロード | Uppy | 3 コンポーネント（`ImageUploadField` / `FileUploadField` / `ImageGalleryField`）で統一。進捗・retry・複数ファイル・並び替えなどの要件を 1 ライブラリでカバーし、進捗 state hook を共有する（doc26 §4.6.4 参照）|
| 6 | カラーピッカー | react-colorful | 2.8KB。事実上の標準 |
| 7 | リピーター | RHF `useFieldArray` | RHF の標準機能でカバー |
| 8 | キーバリュー | RHF `useFieldArray` | リピーターと同じ仕組み |
| 9 | テーブル（インライン編集）| TanStack Table v8 | ヘッドレスで UI 自由。カラム定義が config object 配列でカスタマイズ契約と相性が良い |
| 10 | 表示専用 | shadcn/ui の素のコンポーネント | 特別なライブラリ不要 |
| 11 | チャート | shadcn/ui Chart（Recharts ベース）| Recharts primitive + shadcn/ui の ChartContainer / ChartTooltip / ChartLegend を組み合わせ、`--chart-*` CSS 変数でテーマ統合 |
| 12 | 統計カード | 自前（shadcn/ui Card）| 構造が単純なので外部不要 |
| 13 | 通知・アラート | shadcn/ui Toast（Sonner）/ Alert / Badge | 一時通知は Sonner、静的バナーは Alert、バッジ/カウンターは Badge で使い分け。shadcn/ui は Sonner を標準採用 |
| 14 | オーバーレイ | shadcn/ui Dialog / Sheet / Popover / ContextMenu | Radix ベース |
| 15 | コマンド系 | shadcn/ui Command（cmdk）| コマンドパレット |
| 16 | ツリー操作 | dnd-kit + 自前のツリーロジック | 階層操作（collision 戦略、水平ドラッグの深さ変更判定、親子一括移動）がハンドラ主体で汎用ライブラリに馴染まない |
