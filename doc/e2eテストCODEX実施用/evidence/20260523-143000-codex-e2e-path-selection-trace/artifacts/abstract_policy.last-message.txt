**今回守る制約**
- 管理者ログイン監査では `MFA_REQUIRED_ON_ADMIN_LOGIN` を最優先制約として扱う。根拠: Purpose、`docs/spec/login.md#0002-admin-login-mfa`
- 管理者ログインでは MFA 成功まで管理画面セッションを発行しない。根拠: Source Specs、Conflict Review Item `conflict-login-audit-human-resolution`
- 管理者ログインの成功時と失敗時には `AUDIT_EVENT_REQUIRED` を記録する。根拠: Source Specs、`docs/spec/login.md#0003-audit-event-required`
- 監査イベントには `actor_id`、`source_ip`、`result`、`mfa_result`、`occurred_at` を含める。根拠: Source Specs、`docs/spec/login.md#0003-audit-event-required`
- `AUDIT_EVENT_REQUIRED` の保存先は `audit_log_stream` とする。根拠: Source Specs、`docs/spec/audit.md#0002-audit-storage-boundary`
- `RETENTION_DAYS_FOR_AUTH_LOG` は法務確認待ちのため、Agent が保持日数を決めてはならない。根拠: Source Specs、Core Concept
